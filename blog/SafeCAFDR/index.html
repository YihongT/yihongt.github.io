Problem framing

Most agent safety is reactive. Forecasting agents need safety that looks ahead.

Deep research agents already gather external context and use tools. Forecasting agents increasingly reason over long contexts. But safety work usually checks the current prompt, current output, or current tool call, not the predicted downstream state.

Detailed opportunity map
#1

Predictive safety layer for deep forecasting agents

Combine MoiraiAgent/Dr-CiK style context-gathering with SafePred-style future-risk prediction. The agent forecasts both the target series and the safety/trust state of its reasoning trajectory.

NovelStrong benchmark pathNeeds careful evaluation
Thesis: A forecasting agent should not only ask “what will happen?” but also “will my reasoning path become unsafe, injected, or unreliable before I produce the forecast?”

Safe context selection for forecasting

Final proposal

SafeCAFDR

SafeCAFDR stands for Safe Context-Aware Forecasting Deep Research.

Read refined proposal

Research question

Can a deep-research forecasting agent improve long-horizon forecasting accuracy while proactively predicting and mitigating unsafe or unreliable future reasoning trajectories?

Core hypothesis

Forecasting agents that jointly optimize predictive accuracy, evidence quality, and future risk will outperform context-only agents under realistic and adversarial settings.

Method

  • Context-aware retrieval over news, reports, prior observations, and domain text.
  • Forecasting backbone inspired by time-series foundation models and LLM agents.
  • Predictive risk head that estimates injection, leakage, tool misuse, and hallucination risk.
  • Policy layer that revises context, tools, or explanations before unsafe trajectories unfold.

Evaluation

  • Accuracy on GIFT-Eval-like and domain-specific forecasting tasks.
  • Robustness under adversarial retrieved context.
  • Safety metrics for attack success, leakage, and unreliable evidence use.
  • Ablations for retrieval, risk prediction, and policy intervention.
Execution plan

Path to a publishable paper

A practical route from prototype to paper submission.

Build the baseline

Reproduce a contextual forecasting agent using public time-series tasks and a retrieval-augmented evidence pipeline.

Create adversarial context perturbations

Inject misleading, prompt-injection, and policy-conflicting documents into retrieved evidence.

Add predictive risk modeling

Train or prompt a risk module to forecast whether the current reasoning path will lead to unsafe or unreliable behavior.

Intervene before failure

Use risk estimates to re-rank context, sandbox tools, ask for verification, or suppress unsafe actions.

Compare against strong baselines

Report forecast accuracy, safety robustness, and compute/cost tradeoffs against MoiraiAgent-like and context-only agents.
