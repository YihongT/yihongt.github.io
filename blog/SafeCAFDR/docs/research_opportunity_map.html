Research Opportunity Map

1. ForecastHarnessAudit: Safety Evaluation For Contextual Forecasting Agents

Core idea: adapt HarnessAudit to Dr-CiK-style forecasting agents. Score not only forecast error, but also whether the agent respected evidence provenance, data access boundaries, citation integrity, tool permissions, and inter-agent information flow.

Why it is promising:

  • Low implementation barrier: it can start as an evaluation/benchmark extension before requiring heavy model training.
  • Strong team fit: trajectory audit, harness safety, and information-flow constraints are central to agent trust/security.
  • Clear demand: enterprises care about whether an agent made a forecast by leaking private data, using disallowed tools, or citing poisoned context, not just whether the number was close.

Possible contribution:

  • A benchmark extension over Dr-CiK with safety constraints and adversarial documents.
  • A metric suite: forecast accuracy, evidence recall, distractor citation rate, unauthorized access rate, context leakage rate, and safe-completion rate.
  • A finding likely to be publishable: better forecast accuracy can trade off against worse information-flow or provenance safety.

2. Attack Scaling Laws For Forecasting Agents

Core idea: adapt jailbreak scaling methodology to context attacks on forecasting agents. Instead of single-budget ASR, measure how forecast degradation and safety violations scale with attacker compute, number of poisoned documents, prompt rewrites, or retrieval opportunities.

Why it is promising:

  • Existing forecasting benchmarks usually do not model adversary budget.
  • Dr-CiK already has distractor/citation failure modes that can become adversarial.
  • The scaling-law framing gives a clean evaluation story that is more informative than one attack setting.

Possible contribution:

  • Attack budget curves for contextual forecasting: forecast error, evidence corruption, malicious citation rate, and unsafe tool action rate.
  • Comparison of defenses: retrieval filtering, source trust scoring, CI-aware policies, SafeHarness-style tool control.

3. SafePred For Forecast-Driven Agents

Core idea: use world-model/predictive-guardrail ideas to evaluate actions induced by forecasts. The agent should forecast not only the target time series but also the downstream risk of using certain context, tools, or recommendations.

Why it is promising:

  • This is the cleanest conceptual combination of forecasting and safety.
  • It moves beyond "make the forecast accurate" to "make decisions safely under forecast uncertainty."
  • It could create a new benchmark class: foresight-driven safe agents.

Possible contribution:

  • A risk-to-decision loop for forecasting agents: predict future task utility and future safety risk before using retrieved context or taking tool actions.
  • Counterfactual risk ablations: what happens if the agent follows poisoned context, stale context, or unauthorized context?

4. Robust MoiraiAgent-Style Expert Selection Under Untrusted Context

Core idea: MoiraiAgent selects forecasting experts and applies contextual pre/post-processing. Train or evaluate an expert selector that is robust to poisoned context, misleading metadata, manipulated cross-validation windows, and adversarial anomaly descriptions.

Why it is promising:

  • Directly targets the SOTA path while giving it a trust/security angle.
  • The attack surface is concrete: expert selection, lookback trimming, anomaly removal, and future-effect adjustment.
  • It can start with evaluation before training a new model.

Possible contribution:

  • A robust expert-selection benchmark with clean vs adversarial context.
  • Defense mechanisms: source-aware context features, confidence calibration, constrained transformations, and audit logs.
  • Result target: beat or match MoiraiAgent on clean data while degrading less under adversarial context.

5. Zero-Trust Deep Research Forecasting Agents

Core idea: build a reference architecture for deep-research forecasting agents that treats retrieved context, tools, memory, and generated evidence as untrusted by default.

Why it is promising:

  • It is aligned with Agent Trust and Security.
  • It has a real enterprise demand story: forecasting agents may touch sales, finance, security, medical, climate, supply-chain, or incident data.
  • It can become a systems/security paper if backed by strong benchmark evidence.

Possible contribution:

  • A zero-trust harness for forecasting agents: scoped identities, per-task permissions, sandboxed tools, provenance checks, memory quarantine, rollback, and audit logs.
  • Evaluation on Dr-CiK plus adversarial and permission-constrained variants.

Recommended First Bet

Start with ForecastHarnessAudit. It is the most defensible bridge between your existing Dr-CiK work and the Agent Trust/Security team. It can produce a publishable benchmark/evaluation result quickly, and it creates the substrate for later algorithmic work on SafePred-style defenses or robust MoiraiAgent-style training.

Concrete first experiment:

  1. Take a subset of Dr-CiK tasks.
  2. Add security labels to documents: trusted, untrusted, private, poisoned, stale, and distractor.
  3. Run a MoiraiAgent-like or deep-research-plus-forecaster pipeline.
  4. Log full trajectories: queries, retrieved docs, citations, tool calls, transformations, and final forecasts.
  5. Score both accuracy and safety: forecast error, evidence recall, distractor citation rate, unauthorized evidence use, poisoned evidence use, and unsafe transformation rate.

Demand check:

The demand looks real if the benchmark can show that current high-performing forecasting agents improve accuracy by using context but cannot reliably prove that the context was allowed, fresh, non-poisoned, and causally relevant. That failure is exactly what enterprise agent deployments need to solve before they can trust autonomous forecasting workflows.
