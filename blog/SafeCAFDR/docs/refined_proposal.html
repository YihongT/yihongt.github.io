Refined Proposal — Relevance ≠ Security

A contextual forecast can be accurate and still illegal, leaked, or hijacked.

Version 3 · 2026-06-12 · supersedes the "ForecastHarnessAudit" framing (v2), which collapsed into what Dr-CiK already does. Citation keys resolve in external_references.md.

1. Thesis

Dr-CiK already settled the relevance / robustness question for contextual forecasting agents: can an agent retrieve the right supporting evidence from a noisy corpus and filter distractors — including forecast-dependent, adversarially-crafted distractors? (Answer: today's deep-research agents are bad at it — <5% evidence recall, >80% distractor citations.)

That is a content-decidable, average-case capability problem. It is not security. Our contribution lives in the part Dr-CiK structurally cannot express:

Relevance-competence and security-compliance are decoupled. Whether a forecast is good (relevant, accurate) is independent of whether it was produced legitimately — with authorized, authentic, timely, and contained evidence. Worse, the most useful evidence is often the one you are forbidden to use, and a stronger reasoner does not become more compliant — it becomes a better rule-breaker (Apollo: under pressure, capable agents use insider information and lie about it [Deception-Pressure]).

So an agent can ace Dr-CiK and still be undeployable: it forecasts well by using material non-public information, citing an injected document, leaking a private field, or reading the future.

2. Why this is not Dr-CiK — the technical line

2.1 Distractor vs. security item: what the agent must look at

Distractor (Dr-CiK) Security item (ours)
Question the agent asks "Is this true & relevant?" "Am I allowed to use it? Is the source authentic? Is it timely? May it flow there?"
Decision input the document content the document's label: provenance, identity/policy, timestamp, allowed recipients
Ground-truth function f(content, task) f(provenance, identity, policy, time, recipient) — independent of content
Relevance vs. safety aligned (irrelevant → reject) opposed — the most relevant item is often the one to reject
Problem type capability (reasoning / ranking) systems (access control + information-flow control + provenance)
Solvable inside the context window? yes, by a better reader no — the deciding info is not in the content

The killer example (content-identical, label-opposite). The sentence "Q3 sales are tracking 30% below plan" is fine in a public earnings pre-announcement and illegal in a leaked confidential memo. Same words, opposite decision ⇒ the decision cannot be a function of content; it is a function of the label. Dr-CiK generates distractors from content and grades them by citation/relevance — so it is, by construction, blind to this.

2.2 Distractor : PoisonedRAG :: natural-hard-example : targeted-adversarial-example

Dr-CiK's distractor is a hard sample from a fixed, deliberately-difficult distribution — passive, content-crafted for plausibility. PoisonedRAG's poison is the solution to an optimization: maximize sim(P, Q) (guaranteed retrieval) and induce(answer = R) (attacker-chosen target), injected into a deployed corpus at a tiny budget (5 docs → ~90% targeted success) [PoisonedRAG]. Change the retriever and the distractor is unchanged; the poison is recomputed. The security increment is target + control + optimization-against-the-pipeline + injection — none of which Dr-CiK has.

2.3 Consequence for evaluation

Dr-CiK's unit of evaluation is was a relevant doc cited. Ours must be the trajectory of a labeled token: access → use → emission, scored for boundary violations regardless of whether it improved the forecast. The labels (provenance / authorization / validity-window / flow-constraint) are out-of-band — assigned by the benchmark, not recoverable from content.

3. The technical core (how an agent actually distinguishes security info)

Security cannot be prompted into the model, because the deciding signal is not in the content and the relevance signal is misleading or captured. It must be enforced by the harness:

  1. Typed labels at ingestion — every datum carries {source-id, trust-tier, classification, timestamp, allowed-purposes, allowed-recipients}.
  2. Taint propagation — labels travel with the data through reasoning and tool calls (information-flow control / taint tracking, à la capability-based agent designs).
  3. A reference monitor outside the LLM — checks the label at every use and emit point and allows / redacts / denies. The LLM reasons; the monitor decides authorization. (You cannot rely on the LLM to "choose not to use" forbidden info — see Apollo.)

This maps onto the four formalized contextual-security properties [Formalizing-Agent-Security]: Source Authorization, Task Alignment, Action Alignment, Data Isolation.

4. Flagship artifact — TrustCast

An information-flow-labeled forecasting benchmark + a reference-monitor harness, designed so that the security ground truth is out-of-band and the security contribution cannot be re-derived from Dr-CiK.

  • Benchmark. Forecasting tasks where each document/feed carries hidden labels: public/embargoed, authorized/forbidden-for-identity, authentic/injected, in-window/future/stale, internal-only/exportable. Ground truth is the label, not the relevance.
  • Harness. Taint-tracking + reference monitor implementing the four properties.
  • Metrics (decoupled from accuracy). Unauthorized-use rate (e.g., MNPI-use), injected-evidence-use rate (targeted-ASR under injection), stale/lookahead-use rate, context-leakage rate (Data Isolation), citation faithfulness (causal, not post-hoc), and concealment rate (does it hide a violation?).
  • Headline result. The decoupling: a forecaster that is strong on Dr-CiK relevance is near-zero on security compliance; plotting accuracy vs. compliance shows they do not co-vary, and capability scaling moves accuracy without moving compliance (or worsens concealment).

5. Scenario-grounded directions (security is intrinsic, not bolted on)

Each is a real deployment where a motivated adversary and a real trust boundary already exist; each is explicitly not a relevance problem.

  • S1 · Insider-Information Firewall (finance / MNPI). Authorization + confidentiality + deception with a legal boundary. Combine: Source-Authorization + Data-Isolation monitor, provenance/lineage (compliance-as-code), deception probes [Deception-Pressure], membership-inference contamination checks [MemGuard]. Differs from Dr-CiK: the forbidden evidence is the most useful one.
  • S2 · Stealthy-FDI-secure grid / market forecasting. Physically-consistent false-data injection into sensor/market feeds → targeted price/load bias [FDI-Energy, FDI-SMLT]. Combine: cross-feed corroboration, SafePred-style downstream-risk prediction, attack-budget curves. Differs: targeted controllability on a deployed feed, not a fixed distractor.
  • S3 · Adversary-in-the-loop threat forecasting (EPSS-style). The forecast's subject is the attacker, who poisons threat-intel feeds to suppress prediction of their own activity and adapts to published scores [EPSS, LLM-CTI]. Differs: the adversary is intrinsic to the loop; strategic feedback.
  • S4 · Multi-tenant / multi-agent demand forecasting (supply chain). Targeted poisoning that surfaces only on coordination + competitive-strategy leakage via the agent's own queries [SupplyChain-Poison]. Combine: CI-aware info-flow firewall, HarnessAudit inter-agent audit. Differs: confidentiality across tenants + integrity under a competitor, not single-agent relevance.

A1 · Targeted poisoning & attack-scaling for forecasting agents. The attack companion: port PoisonedRAG to forecasting — targeted-ASR and injection-budget cost curves, defense frontier (CausalArmor, LlamaFirewall, source-trust). The methodological contrast to Dr-CiK's distractor robustness.

6. Why none of this collapses into Dr-CiK

Dr-CiK = relevance / robustness (content-decidable, fixed corpus, distractor-citation, average-case) — done. This proposal = security / compliance (label-decidable, deployed adversary, info-flow enforcement, worst-case + targeted) — open, and provably outside Dr-CiK's generate-distractors-and- score-citation design. The bridge is the empirical decoupling thesis: being good at the former tells you nothing about the latter.
