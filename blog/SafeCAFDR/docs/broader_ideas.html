Broader Ideas

These are adjacent research directions around the SafeCAFDR theme. They can become ablations, companion benchmarks, or follow-up papers.

1. Safe context selection for forecasting

Rank retrieved evidence using both predictive relevance and safety risk. The key question is whether a forecasting agent can reject malicious or misleading context without losing useful signal.

  • Inputs: news, reports, domain notes, prior observations.
  • Risks: prompt injection, stale evidence, leakage, adversarial framing.
  • Outputs: safer evidence sets and calibrated forecast explanations.

2. Forecasting-agent security benchmark

Create adversarial forecasting tasks where injected context tries to change the horizon, tools, data interpretation, or explanation. This is especially useful for evaluating deep-research agents that depend on external documents.

3. Future-risk-aware tool policy

Instead of allowing tools whenever the current step looks safe, predict whether the next few steps will create unsafe states. The policy can then require verification, sandboxing, or reduced privileges.

4. Explanation reliability for long-horizon forecasts

Evaluate whether generated explanations cite reliable evidence and whether the explanation changes under adversarial context. This can connect forecasting accuracy with trustworthiness.

5. Multi-agent forecasting debate with safety monitors

Run multiple forecasting agents with different evidence views and use a safety monitor to detect collusion, overconfidence, or evidence poisoning.
