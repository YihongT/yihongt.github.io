Broader Ideas — Agent Trust & Security (non-forecasting)

For pitching to the group beyond the forecasting thread. Each idea reuses primitives the team already cares about — trajectory auditing, information flow, provenance, predictive guardrails, attack-budget scaling — on a general agent substrate. Citation keys resolve in external_references.md. Each entry follows the same four-facet structure: Motivation · Expected contribution · Experiment / application · How it differs.

The forecasting ideas (F1–F5) live in research_opportunity_map.md and refined_proposal.md; the same four-facet structure is mirrored on the website.

B1 · Provenance-Faithful Agents — certifying why an agent acted

  • Motivation. Correctness ≠ faithfulness: up to 57% of RAG citations are post-rationalized [Cite-Faithfulness] — the model decides first and attaches a plausible source afterward. An agent that "explains" its decision may be confabulating the cause, so trust requires proving the cited evidence actually drove the action.
  • Expected contribution. A counterfactual audit (ablate the cited evidence → does the action change?) that emits a per-trajectory faithfulness certificate, plus a benchmark and metric extending HarnessAudit's execution-fidelity axis.
  • Experiment / application. Run on tool-using QA, coding, and research agents; report the faithfulness-vs-correctness gap and certificate accuracy. Deployable as a trust badge / regulatory audit signal for any agent.
  • How it differs. [Cite-Faithfulness] studies QA citations statically; B1 makes it causal and agentic over the trajectory. [CausalArmor] uses causal attribution to defend against injection; B1 repurposes attribution to certify provenance.

B2 · Attack Scaling Laws for Indirect Prompt Injection

  • Motivation. IPI is evaluated at a single attacker setting [AgentDojo, IPI-Competition]; we lack predictive scaling curves, so defenders cannot compare on a common budget axis.
  • Expected contribution. Attack-success and unsafe-tool-call rate vs attacker budget (injected docs, rewrite compute, retrieval ops) across agent frameworks; saturating-curve fits; a defense efficiency frontier (CausalArmor, LlamaFirewall, SecAlign) ranked by compute-to-break.
  • Experiment / application. Sweep injection budget across AgentDojo-style suites; fit curves; report which formal property (per [Formalizing-Agent-Security]) breaks first. Application: red-team budgeting and procurement SLAs.
  • How it differs. Jailbreak-Scaling targets chat refusals on a FLOPs axis; B2 targets agent IPI with tool-call outcomes and property-level breakage.

B3 · Temporal-Validity Guardrail — agents that know when context is stale

  • Motivation. Agents can't reliably tell when memories/evidence are no longer valid [STALE]; stale documents degrade RAG [HoH]; lookahead leakage inflates apparent skill [Lookahead-LLM]; the 2026 trustworthy-agent survey flags temporal staleness as a core evaluation pitfall [Trustworthy-Agentic-Survey].
  • Expected contribution. A validity-window + supersession-detection mechanism and a benchmark of time-conflicting evidence, scored by stale-use rate and post-hoc leakage rate; an abstain-when- invalid policy.
  • Experiment / application. Inject superseded / future-dated documents; measure stale-use before vs after the guardrail; integrate into memory-using assistants. Application: long-horizon assistants, news/financial agents, anything over evolving knowledge.
  • How it differs. T-GRAG / SmartVector add temporal awareness to retrieval ranking; B3 is a guardrail that detects invalidity and abstains, evaluated as a safety property and paired with predictive (SafePred) framing.

B4 · CI-Aware Multi-Agent Information-Flow Firewall

  • Motivation. SOTA multi-agent systems leak 35–51% of sensitive fields across handoffs [AgentLeak, MAGPIE]; data/instruction separation is provably insufficient [PromptInj-CI]; HarnessAudit finds risk concentrates in inter-agent transfer.
  • Expected contribution. A runtime monitor grounded in Contextual Integrity that labels per-field flow norms and blocks/redacts inappropriate inter-agent transfers; a privacy↔utility frontier evaluation; direct realization of the Data Isolation property [Formalizing-Agent-Security].
  • Experiment / application. Run on AgentLeak/MAGPIE-style tasks; measure leakage reduction vs utility loss; ablate flow rules. Application: enterprise multi-agent systems handling confidential fields.
  • How it differs. AgentLeak/MAGPIE measure leakage; 1-2-3-Check uses multi-agent reasoning for privacy; B4 is an enforced information-flow firewall (mechanism + property compliance), not just measurement or prompting.

B5 · Predictive Guardrails Beyond CUAs

  • Motivation. SafePred targets computer-using agents; reactive guardrails miss delayed harm. Many high-impact agents (coding, devops, research) have delayed-consequence failures — e.g., a refactor that silently breaks the audit trail six steps later.
  • Expected contribution. Generalize world-model future-risk prediction to general tool/coding agents; a delayed-consequence benchmark; step-level intervention plus task-level re-planning.
  • Experiment / application. Construct tasks where harm emerges N steps later; measure detection / prevention vs a reactive baseline. Application: autonomous coding/devops agents.
  • How it differs. SafePred = CUAs/GUI actions; B5 = general tool/coding agents with code-and-state consequences, complementing LlamaFirewall's reactive CodeShield with predictive foresight.

B6 · Zero-Trust Agent Harness + Conformance Suite

  • Motivation. Zero-Trust-for-AI-Agents [ZeroTrust] is a framework/whitepaper, not a measured system; LlamaFirewall is production but partial; no standard conformance benchmark exists; 2026 MCP tool-poisoning incidents show the need [MCP-Threat-Model].
  • Expected contribution. A reference harness (scoped identities, least-privilege tools, sandboxing, memory quarantine, provenance checks, rollback, audit logs) plus a conformance benchmark scoring any harness against these controls — unifying SafeHarness and HarnessAudit.
  • Experiment / application. Implement the harness; score it and baselines (LlamaFirewall, vanilla) on the conformance suite under MCP-poisoning + IPI attacks. Application: enterprise procurement / certification of agent platforms.
  • How it differs. SafeHarness proposes lifecycle layers; HarnessAudit audits trajectories; B6 is the deployable zero-trust architecture plus a conformance standard tying them together with identity and least-privilege.

How they relate to the forecasting bet

ForecastHarnessAudit is the vertical (one high-stakes domain, deep). B1–B6 are the horizontals (one primitive each, broad). The forecasting bench produces the trajectories and metrics that B1, B2, B3, and B5 can reuse — so the vertical de-risks the horizontals.
